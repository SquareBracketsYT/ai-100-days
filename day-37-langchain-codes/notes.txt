LCEL:
LangChain Expression Language
- Declarative way to connect LangChain components into reusable pipelines

Without LCEL:
prompt = prompt_template.invoke(data)
response = model.invoke(propmp)
final = parser.invoke(response)

With LCEL:
chain = prompt | model | parser


Runnable:
- Everything in LCEL is a Runnable

RunnableSequence:
chain = prompt | model | parser

from langchain_core.runnables import RunnableSequence
chain = RunnableSequence([coding_prompt, get_model(), output_parser])

Context:
Information that models receives along with the current request to generate a response


Stateless vs Stateful
Stateless- Every request is independent
Stateful- Every new request includes previous message


Message types in LangChain
- HumanMessage - User Input
- AIMessage - Model Response
- SystemMessage - Defines behaviour


Context Window
- Every model has a maximum amount of context it can process

Context includes:
- System prompt
- Previous Conversation
- Current question


RunnableLambda -> preprocess the input
MessagePlaceholder -> inject conversation history
Conversation memory -> maintain the context
Context management -? control what the model sees
